## Core Principles

- **Protected Diversity**: Different mental models should have meaningful space to operate according to their own logic.
- **Stewardship Governance**: Long-term care for the codebase and future maintainers takes precedence over short-term gains.
- **Constitutional Constraints**: Foundational rules are harder to change than day-to-day decisions.
- **Discrimination & Quality**: The system must reliably distinguish valuable results from noise.
- **Real Stakes**: Influence and survival are tied to demonstrated outcomes.
- **Sustainable Development**: Work proceeds through focused sprints with clear, narrow deliverables.

## Architecture Overview

Forge Federation is structured in three primary layers:

- **Stewardship Governance Layer** — The constitutional layer. Defines long-term values and hard constraints. Uses moots (deliberative processes) for changing system-level rules.
- **Spider Web Discrimination Layer** — Cross-cutting quality and filtering capability. Focuses on multi-stage evaluation, selective response, and preventing low-quality output from propagating.
- **Territories (Mental Model Ecosystems)** — Semi-sovereign groups of agents operating under different reasoning paradigms. They propose and critique changes while remaining subject to constitutional constraints.

## Current Status & Expectations

This project is in early development. We have strong conceptual foundations from the original PrizmForge work and observations from live systems like mydeadinternet.com, but the integrated architecture is still being built.

**Development expectations:**
- Work happens in short, focused sprints rather than rigid long-term plans.
- We prioritize working prototypes over architectural perfection in early stages.
- Scope is deliberately narrowed per sprint so meaningful progress can be made consistently.
- Existing permissive-license frameworks (such as CrewAI and LangGraph) are used heavily.
- The work API budget is reserved primarily for high-value reasoning and evaluation.
- Local hardware is used for volume, persistence, and lower-stakes background work.

## Technology & Constraints

- Preference for permissive open-source licenses
- Focus on practical, governed code improvement
